# 19.5 Zustand vs 其他方案


## 1. vs Redux (Toolkit)

| 特性 | Zustand | Redux Toolkit |
| :--- | :--- | :--- |
| **心智模型** | 极简，Hooks 优先 | Flux，单一状态树 |
| **Boilerplate** | 极少 | 中等 (Slice 简化了很多) |
| **Provider** | 不需要 | 需要 `<Provider>` |
| **DevTools** | 支持 | 原生支持极佳 |
| **适用场景** | 中小型应用，敏捷开发 | 大型应用，团队协作 |

## 2. vs Context API

| 特性 | Zustand | Context API |
| :--- | :--- | :--- |
| **性能** | 自动优化，Selector 避免不必要渲染 | 无论用不用，Provider 变了子组件全渲染 (需配合 memo) |
| **外部访问** | 可以 (getState/setState) | 困难 (只能在组件内) |
| **使用难度** | 简单 | 简单 |

## 3. vs Jotai / Recoil

- **Zustand** 是 **Flux** 模式的简化版，状态是中心化的（Store）。
- **Jotai / Recoil** 是 **Atomic** 模式，状态是分散的（Atom），更适合状态之间依赖复杂、图状结构的场景（如图形编辑器）。
