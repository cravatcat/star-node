# 80. 不同路径

## 题目描述

一个机器人位于一个 `m x n` 网格的左上角 （起始点在下图中标记为 “Start” ）。

机器人每次只能向下或者向右移动一步。机器人试图达到网格的右下角（在下图中标记为 “Finish” ）。

问总共有多少条不同的路径？

**示例 1：**

```text
输入：m = 3, n = 7
输出：28
```

**示例 2：**

```text
输入：m = 3, n = 2
输出：3
解释：
从左上角开始，总共有 3 条路径可以到达右下角。
1. 向右 -> 向下 -> 向下
2. 向下 -> 向下 -> 向右
3. 向下 -> 向右 -> 向下
```

## 实现思路

这是一个经典的动态规划问题。

定义 `dp[i][j]` 为到达位置 `(i, j)` 的路径数量。
由于只能向下或向右移动，所以到达 `(i, j)` 只能从 `(i-1, j)` 或 `(i, j-1)` 过来。
状态转移方程：`dp[i][j] = dp[i-1][j] + dp[i][j-1]`。

初始条件：
- 第一行只能一直向右走，所以 `dp[0][j] = 1`。
- 第一列只能一直向下走，所以 `dp[i][0] = 1`。

**空间优化**：
观察状态转移方程，`dp[i][j]` 只依赖于上方和左方的值，因此可以将二维数组优化为一维数组。
`dp[j] = dp[j] + dp[j-1]`，其中等号右边的 `dp[j]` 代表上一行的值（相当于 `dp[i-1][j]`），`dp[j-1]` 代表当前行左边的值（相当于 `dp[i][j-1]`）。

## 代码实现

```typescript
export function uniquePaths(m: number, n: number): number {
  // 使用一维数组进行优化
  const dp = new Array(n).fill(1);

  for (let i = 1; i < m; i++) {
    for (let j = 1; j < n; j++) {
      dp[j] += dp[j - 1];
    }
  }

  return dp[n - 1];
}
```

## 复杂度分析

- **时间复杂度**：O(m * n)。
- **空间复杂度**：O(n)，使用了一维数组进行优化。
