# 回文链表 (Palindrome Linked List)

## 1. 题目描述

给你一个单链表的头节点 `head` ，请你判断该链表是否为回文链表。如果是，返回 `true` ；否则，返回 `false` 。

**示例 1:**

![pal1linked-list](https://assets.leetcode.com/uploads/2021/03/03/pal1linked-list.jpg)

> **输入:** head = [1,2,2,1]  
> **输出:** true

**示例 2:**

![pal2linked-list](https://assets.leetcode.com/uploads/2021/03/03/pal2linked-list.jpg)

> **输入:** head = [1,2]  
> **输出:** false

**进阶：** 你能否用 $O(n)$ 时间复杂度和 $O(1)$ 空间复杂度解决此题？

## 2. 实现思路

要以 $O(1)$ 的空间复杂度解决这个问题，我们不能简单地将链表元素复制到数组中然后检查。我们需要直接在链表上操作。

核心步骤如下：

1.  **找到前半部分链表的尾节点**：我们可以使用 **快慢指针**。快指针每次走两步，慢指针每次走一步。当快指针到达末尾时，慢指针正好位于中间位置。
2.  **反转后半部分链表**：从中间位置开始，将后半部分链表进行反转。
3.  **比较两部分链表**：将前半部分链表和反转后的后半部分链表进行节点值比较。
4.  **恢复链表（可选但推荐）**：为了不破坏原链表结构，比较完成后再次反转后半部分链表，将其恢复原状。

## 3. 详细步骤

1.  如果链表为空或只有一个节点，直接返回 `true`。
2.  使用 `fast` 和 `slow` 指针寻找链表中点。
    *   初始化 `fast = head`, `slow = head`。
    *   循环条件：`fast.next` 和 `fast.next.next` 都不为空。
    *   `fast` 走两步，`slow` 走一步。
3.  `slow` 此时指向前半部分的尾节点。反转 `slow` 之后的链表，得到后半部分的新头节点 `secondHalfStart`。
4.  比较回文：
    *   `p1` 指向 `head`，`p2` 指向 `secondHalfStart`。
    *   循环遍历 `p2`，如果 `p1.val !== p2.val`，则不是回文。
5.  （可选）再次反转后半部分，恢复链表结构。
6.  返回结果。

## 4. 代码实现 (TypeScript)

```typescript
/**
 * Definition for singly-linked list.
 * class ListNode {
 *     val: number
 *     next: ListNode | null
 *     constructor(val?: number, next?: ListNode | null) {
 *         this.val = (val===undefined ? 0 : val)
 *         this.next = (next===undefined ? null : next)
 *     }
 * }
 */

export function isPalindrome(head: ListNode | null): boolean {
    if (head === null || head.next === null) {
        return true;
    }

    // 1. 找到前半部分链表的尾节点
    const firstHalfEnd = endOfFirstHalf(head);
    // 2. 反转后半部分链表
    const secondHalfStart = reverseList(firstHalfEnd!.next);

    // 3. 判断是否回文
    let p1: ListNode | null = head;
    let p2: ListNode | null = secondHalfStart;
    let result = true;
    while (result && p2 !== null) {
        if (p1!.val !== p2.val) {
            result = false;
        }
        p1 = p1!.next;
        p2 = p2.next;
    }

    // 4. 还原链表 (保持良好习惯，不破坏原有结构)
    firstHalfEnd!.next = reverseList(secondHalfStart);
    return result;
}

const reverseList = (head: ListNode | null): ListNode | null => {
    let prev: ListNode | null = null;
    let curr: ListNode | null = head;
    while (curr !== null) {
        let nextTemp: ListNode | null = curr.next;
        curr.next = prev;
        prev = curr;
        curr = nextTemp;
    }
    return prev;
}

const endOfFirstHalf = (head: ListNode): ListNode | null => {
    let fast: ListNode | null = head;
    let slow: ListNode | null = head;
    while (fast!.next !== null && fast!.next.next !== null) {
        fast = fast!.next.next;
        slow = slow!.next;
    }
    return slow;
}
```

## 5. 复杂度分析

*   **时间复杂度**: $O(n)$。其中 $n$ 是链表的节点数。我们需要寻找中点（遍历一次）、反转后半部分（遍历一半）、比较（遍历一半）、恢复（遍历一半）。总体是线性的。
*   **空间复杂度**: $O(1)$。我们只使用了几个指针变量，是在原链表上修改，没有使用额外的线性空间。
